# Project Management

This repository uses GitHub Issues to track work.

## Issue types
- `feature`: a new user-facing or technical capability
- `bug`: something broken or behaving incorrectly
- `improvement`: refinement to UX, performance, quality, structure, or content

## Recommended labels
### Type
- `feature`
- `bug`
- `improvement`

### Priority
- `priority: high`
- `priority: medium`
- `priority: low`

### Area
- `area: landing-page`
- `area: dashboard`
- `area: admin`
- `area: employee`
- `area: claims`
- `area: auth`
- `area: payments`
- `area: backend`
- `area: seo`
- `area: design`

### Severity
- `severity: critical`
- `severity: major`
- `severity: minor`

## Suggested workflow
1. Create an issue with the correct template.
2. Add labels for type, priority, and area.
3. Add the issue to your GitHub Project board.
4. Move the issue through: `Backlog -> Planned -> In Progress -> Review -> Done`.
5. Link pull requests to issues with `Fixes #issue_number`.

## SafePhone usage
Use this repository primarily for frontend and product-facing work such as:
- landing page
- dashboard UI
- design improvements
- customer flows
- employee/admin screens

Keep backend/API/infrastructure issues in the backend repository when they are implementation-specific.
